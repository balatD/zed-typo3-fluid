# TYPO3 Fluid for Zed

A [Zed](https://zed.dev) extension providing language support for the
[TYPO3 Fluid](https://docs.typo3.org/permalink/t3coreapi:fluid) templating
language. It is a port of the
[FriendsOfTYPO3/vscode-fluid-language](https://github.com/FriendsOfTYPO3/vscode-fluid-language)
VS Code extension.

> **Status: highlighting + snippets**, powered by a dedicated
> [`tree-sitter-fluid`](../tree-sitter-fluid) grammar. ViewHelper autocomplete,
> inline documentation and live template analysis (a language server) are not yet
> implemented. See [Roadmap](#roadmap).

## Why this isn't a straight port

Zed and VS Code have incompatible foundations, so the source extension could not
be copied file-for-file:

| Concern | VS Code extension | Zed |
| --- | --- | --- |
| Syntax highlighting | TextMate grammar (`.tmLanguage`) | **Tree-sitter** only ‚Üí new grammar |
| ViewHelper completion / docs | VS Code **HTML Custom Data** | no equivalent ‚Üí needs an LSP |
| Live analysis | TS client runs `fluid`/`typo3` PHP binary | needs an LSP |
| Snippets | `snippets/*.json` | same format ‚úÖ |
| File detection | globs over `Resources/Private/**` dirs | `path_suffixes` + `first_line_pattern`, **no globs** |

Highlighting is driven by a purpose-built **`tree-sitter-fluid`** grammar (in the
sibling `tree-sitter-fluid/` repo) that extends tree-sitter-html and actually
parses Fluid syntax ‚Äî so `{‚Ä¶}` expressions, inline ViewHelpers, operators, casts
and arrays are tokenized, not just the surrounding HTML.

## Features

- Full HTML highlighting (elements, attributes, comments, entities, embedded
  CSS/JS).
- Fluid **ViewHelper tags** ‚Äî `<f:if>`, `<f:format.raw>`, `<v:‚Ä¶>`, `<core:icon>`,
  custom `vendor:‚Ä¶` namespaces ‚Äî as functions.
- Fluid **`{expressions}`**, tokenized everywhere (text **and** attribute values):
  - object accessors `{user.name}` ‚Üí variables,
  - inline ViewHelpers & pipelines `{foo -> f:format.raw()}` ‚Üí functions,
  - named arguments / array keys ‚Üí properties,
  - operators, ternaries, `as` casts, booleans/`_all`, numbers, strings.
- **`{namespace ‚Ä¶}`** declarations, `xmlns:f`, `data-namespace-typo3-fluid`.
- **Dynamic tag names** `<{headline}>‚Ä¶</{headline}>`.
- 26 **snippets** for core and TYPO3 ViewHelpers (`f:for`, `f:if`, `f:translate`,
  `f:image`, `f:render.*`, ‚Ä¶).

### Known limitations

- **File detection.** Zed `path_suffixes` can't glob TYPO3's
  `Resources/Private/Templates/**` directories. Detection is therefore: the
  explicit Fluid suffixes (`*.fluid.html`, `*.fluid`, ‚Ä¶) **plus** a
  `first_line_pattern` that sniffs `{namespace`, `data-namespace-typo3-fluid`,
  `xmlns:f`, or a `<f:‚Ä¶>`/`<x:y>` tag on line 1. Templates that open with plain
  HTML still need a manual language switch (command palette ‚Üí language selector).
- **Grammar edge cases** (localized, non-fatal ‚Äî the rest of the file still
  highlights): same-quote nesting inside an attribute value
  (`value='{ ‚Ä¶ 'x': ‚Ä¶ }'`) and the interior of string literals are not fully
  tokenized.

## File detection

Auto-detected suffixes: `*.fluid.html`, `*.fluid.htm`, `*.fluid.txt`, `*.fluid`.

## Install (development)

1. Clone this repo.
2. In Zed: **Extensions ‚Üí Install Dev Extension**, then select this directory.
3. Open a file from `test/fixtures/` (e.g. `tag-viewhelpers.fluid.html`) and
   confirm highlighting. Type `f:for` and press <kbd>Tab</kbd> to expand a snippet.
4. For logs, launch Zed with `zed --foreground`.

## Roadmap

- **Phase 3 ‚Äî Language server.** A standalone Fluid LSP (spawned by a small
  Rust/WASM glue layer via `language_server_command`) providing:
  - ViewHelper tag + argument **completion**,
  - inline ViewHelper **documentation** (hover),
  - **diagnostics** from the project's `fluid`/`typo3` binary (with DDEV support).

  The ViewHelper dataset generated by the source extension's `fluid/` PHP project
  (`schema_TYPO3_CMS_*_ViewHelpers.json`) is the intended data source for
  completion and hover ‚Äî no regeneration required.
- **Optional ‚Äî dedicated `tree-sitter-fluid` grammar** to fully tokenize `{‚Ä¶}`
  expression interiors.
